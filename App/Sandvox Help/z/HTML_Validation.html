HTML Validation

Sandvox allows you to validate the HTML of any page or object. This is mainly useful for pages which contain your own custom HTML content (such as Raw HTML pages or objects). The validation service is provided by w3.org.

To validate a page's HTML:

  1. Ensure an Internet connection is available.
  2. From the View menu choose "Validate HTML."

Validating an entire HTML page is useful for ensuring that the page as a whole has been built properly, taking all the pieces (injected code, the HTML templates built by Sandvox, Raw HTML objects) into account. It's most useful for checking that your Code Injection is OK. But if there are Raw HTML objects, then it's better to validate just that object, so that it is easier to identify any problems, separate from the rest of the page's contents.

If the page fails validation, Sandvox will display the results in a new window. It is then up to you to correct the issues causing the validation to fail.

Please note that it is possible for an HTML page to fail validation, yet be still perfectly usable for all or most browsers. Validation is an important tool, but in practice, it is not critical that your website contain completely valid HTML.
