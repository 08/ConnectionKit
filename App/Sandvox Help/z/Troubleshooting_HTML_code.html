Troubleshooting HTML code

There are two common reasons that HTML code added to a Sandvox site does not work as planned:

  1. Check that you have correctly purchased and registered the Pro edition of Sandvox. Only Sandvox Pro supports HTML additions to pages and pagelets; the standard version does not.
  2. Check your code. A missing forward slash, an omitted tag, or an incorrect tag can create a variety of problems.


As much as the staff of Karelia would like to help you make your site exactly what you envision, we cannot proofread HTML, provide advice or guidance on how to use HTML to accomplish a particular goal, or offer an HTML tutorial.

The good news is there are many sites that do provide such guidance, and whether you're new to HTML or an old-timer who can't remember a particular tag, we encourage you to review the following sites when troubleshooting HTML:

  • W3Schools HTML Tutorial
  • The HTML Code Tutorial
  • EchoEcho.com HTML Tutorial
  • The Webmonkey HTML Cheatsheet

I can't enter non-HTML code properly

You may sometimes find that custom non-HTML code (such as scripts) entered into Sandvox's editing window does not appear in the web view, and then does not remain in the editing window.

This is because there are two possible ways of entering custom HTML code into Sandvox; a raw HTML page/pagelet, or by using the "Edit Raw HTML" menu command on a block of text.

With the latter, you are limited in the code that can be entered. Editing is performed by Sandvox directly manipulating the web view's DOM tree. This means that code entered is interpreted as best as possible and then saved back into the document. As a result, if you enter non-valid HTML code such as PHP directives or other scripts, Sandvox may not be able to accept it.

The solution is to use a raw HTML page or pagelet which has no such limitation. Any code entered is exactly preserved so you can use scripts and any other non-standard HTML.

Third party provided code does not appear correctly

A common use for adding custom HTML code to a Sandvox site is to incorporate HTML provided by another website or service. In some cases this code will actually check the URL of the site it is being loaded from and behave accordingly (e.g. Weather.com forecast data). This can result in code that appears correctly on your published website, but not within the Sandvox interface. At present, the only solution is to test such code online using your website.

Images in my HTML code appear as blue question marks

One common issue is that an <img> tag does not appear correctly in the Web View. Instead of seeing the image you expect, a blue question mark appears:

This could be because:

  • "Show live data feeds" is disabled in the Preferences. Sandvox will not load any content from the Internet with this preference set, but images will appear correctly on the published site.
  • You have not correctly specified the source of the image. Try opening the source URL in your web browser to check it is correct.
