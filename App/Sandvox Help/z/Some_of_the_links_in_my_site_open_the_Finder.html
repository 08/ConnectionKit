Some of the links in my site open the Finder

Sandvox offers the ability to export a site to a folder on your computer. This allows you to browse the site in a web browser before publishing it, or upload it using an FTP client.

If you attempt to view your exported website by directly opening the files using the Finder, you may notice an issue when following links that point to a collection (folder). When clicked, the link opens a folder in the Finder rather than the collection's associated "index.html" page in the browser. This is the normal behavior of your browser when viewing pages directly from the file system. While this can be annoying, this behavior does not occur once the site is published and the pages are viewed from an "http://..." address in your web browser.

You can work around this by changing a hidden preference; see "Force paths to collections to end in index file" on that page to force links to collections to end with "index.html". This is useful if you are building a website that is designed to be viewed from the Finder, e.g. a CD-ROM presentation.
