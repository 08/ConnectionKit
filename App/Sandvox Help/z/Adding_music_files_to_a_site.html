Adding music files to a site

There are a number of different ways to add music files to a site. Which method you choose depends on how you want visitors to listen to them.

Downloads

If you wish for visitors to simply download the music file, you need to create a File Download and link to that.

  1. Click the "New" item in the toolbar and choose "Choose..." option at the bottom.
  2. Navigate to the file that you want to add in the Finder window.
  3. Click the "Insert" button to select the music file.
  4. On another page, create a link to the File Download you just added.

By adding the music as a File Download, it will either download to their computer or play using the media player of their choice.

Within a Page

A music file can also be placed within a page's main content area to allow visitors to listen to the audio while reading the rest of the page. To do so requires a Media Object.

  1. Make a new page or select an existing one of these in the Site Outline.
  2. Locate the music file in the Media Browser, Finder or another application.
  3. Drag and drop the file onto the Sidebar or main content area of the page.

Alternatively, you could keep the object in the Sidebar. If it appears on a Collection page, you can use this to have the music available on all pages that are part of the Collection as well.

Background Music

You can easily add background music to a single page or a whole collection of pages in your site.

To add background music:

  1. Add a new Media Placeholder object to the pages you wish to have the background music. You can add the Media Placeholder to a Collection page (such as the Home Page) in order for the music to play on every page in the collection.
  2. Select the new media object.
  3. Open the Object Inspector and set the file info to use the appropriate file for background music.

If you have added the Media object to a Collection page in the sidebar, it will show on all of the pages of that collection that are also showing the sidebar.
To have the background music automatically play when the page is opened, check the "Automatically Play" box in the Object Inspector.
You may wish to have the background music play continuously. To do so, check the "Loop" box in the Object Inspector.
Note: Many people prefer not to have a website start playing music automatically.

File Types

The best format for audio is to provide a .wav or .mp3 file.
iOS does have some restrictions on the specific encoding in an .mp3 file, so it is important to test the audio on an iPhone/iPad.
Other formats like .m4a, .aiff, .au, .mov (QuickTime audio-only), etc. may be playable, and show up in Sandvox - but many browsers are not going to be able to play them.
The inspector will show a warning when these partially-supported file formats have been chosen.

Protected audio and video files, such as a track bought at the iTunes store, are not going to work either; they are licensed only for the purchaser.
