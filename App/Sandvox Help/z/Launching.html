Launching

When you launch Sandvox, it goes through a number of standard functions:

Preflight Check

We have a simple wrapper application that checks to make sure you are running a version of Mac OS X that supports Sandvox. (This wrapper also sets up some internals to allow Sandvox to use the newer version of Apple's WebKit than comes built into Mac OS X.)

Phone Home

Sandvox normally "phones home" to Karelia Software (unless you have this turned off in preferences).

It transmits the following information:

  • The version of Sandvox you are running, to check if there is a new version
  • The version of Mac OS that you are running
  • The first four languages you have chosen in your system, so that the server's response can be localized.

Our server returns information about any newer version of Sandvox available, plus a hint on the modification date of the Sandvox weblog, so that you can be alerted to new Sandvox news from within the application.

Crash Reporter

Sandvox checks to see if it had recently crashed, so that a report of the problem can be sent to Karelia.

Load Plugins

Sandvox searches in the Library → Application Support → Sandvox folder for any additional plugins (functionality or additional designs) you may have installed.

Open Recent Documents

Sandvox automatically reopens any documents that were open when you last quit. If no recent documents can be found, Sandvox will ask you if you wish to open an existing site or create a new one. This dialog also appears if you activate Sandvox (say, from the dock) when no document windows are open.

You can disable this preference using Sandvox's Hidden Preferences.
